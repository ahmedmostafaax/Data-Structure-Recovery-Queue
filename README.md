## نظرة عامة

الطابور (Queue) هو هيكل بيانات يتبع مبدأ "أول من يدخل أول من يخرج" (FIFO - First In First Out)، مثل طابور الانتظار في البنك أو السوبر ماركت. هذا التنفيذ يستخدم كائن (Object) بدلاً من المصفوفة لتحقيق أداء مثالي.

## المميزات

- ✅ **عمليات سريعة**: O(1) للإضافة والإزالة
- ✅ **بدون تحويل مصفوفة**: استخدام مؤشرات head/tail
- ✅ **كفاءة في الذاكرة**: يخزن العناصر النشطة فقط
- ✅ **معالجة الأخطاء**: التحقق من الحالات الفارغة
- ✅ **واجهة بسيطة**: دوال واضحة وسهلة الاستخدام

## لماذا هذا التنفيذ؟

التنفيذ التقليدي باستخدام المصفوفات يعاني من مشكلة أداء:

```javascript
// تنفيذ غير فعال باستخدام المصفوفة
class ArrayQueue {
    dequeue() {
        return this.items.shift(); // O(n) - تنقل كل العناصر!
    }
}

// تنفيذ فعال باستخدام الكائن (هذا المشروع)
class ObjectQueue {
    dequeue() {
        const item = this.items[this.head];
        delete this.items[this.head];
        this.head++; // O(1) - بدون تنقل!
        return item;
    }
}
